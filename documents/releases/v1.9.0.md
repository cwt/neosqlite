# NeoSQLite v1.9.0 Release Notes

## TL;DR

- **New Core Feature**: **ACID Transactions** via `ClientSession` API with full PyMongo 4.x parity.

## Overview

NeoSQLite v1.9.0 is a major milestone focusing on **API Completeness** and **Optimization Expansion**. While the three-tier architecture was established in earlier versions, v1.9.0 significantly broadens its reach, bringing SQL-level performance to many of the most complex MongoDB operators and stages.

This release also introduces foundational new capabilities like **ACID Transactions** and **Modern GridFS**, making NeoSQLite a truly production-ready, drop-in replacement for PyMongo in embedded environments.

## New Core Feature: ACID Transactions (`ClientSession`)

For the first time, NeoSQLite supports full **ACID transactions** via the `ClientSession` API:
- **PyMongo 4.x Parity**: Implements `start_transaction()`, `commit_transaction()`, and `abort_transaction()`.
- **Callback Support**: `with_transaction()` provides reliable transactional execution.
- **Native Integration**: Leverages SQLite's `SAVEPOINT` mechanism for robust, nested transaction support within your application logic.

## Aggregation: Advanced Stages & Tier-1 Expansion

v1.9.0 brings a massive expansion to the aggregation framework, with a focus on moving logic from Python (Tier-3) to SQL (Tier-1):

### 1. New High-Impact Stages
- **`$setWindowFields`**: Comprehensive window function support including `$rank`, `$denseRank`, `$top`, and `$bottom`.
- **`$graphLookup`**: Recursive hierarchical searches implemented via optimized Recursive CTEs.
- **`$fill`**: Intelligent data filling (constant or `locf`) for sequential datasets.
- **Streaming `$facet`**: Parallel sub-pipeline processing with streaming results to handle large multi-faceted aggregations efficiently.

### 2. Tier-1 (SQL) Optimization Expansion
Many operators that previously relied on Python fallback now run directly in SQL:
- **Set Operators**: All 7 operators (`$setUnion`, `$setIntersection`, etc.) are now Tier-1 optimized.
- **String Power**: `$split` (via recursive CTE), `$replaceAll`, and `$replaceOne` now run in SQL.
- **Variable Scoping**: `$let` is now fully SQL-tier optimized via variable inlining.
- **Accumulators**: `$addToSet` and `$stdDev` (Pop/Samp) now have Tier-1 SQL implementations.

## Security: Native `$jsonSchema` Validation

v1.9.0 introduces robust JSON Schema support:
- **Query Validation**: Use `$jsonSchema` within `find()` or `$match` stages.
- **Write Validation**: NeoSQLite can now translate `$jsonSchema` into native **SQLite CHECK constraints**, providing database-level enforcement of your document structure.

## Performance: Native JSONB & Memory Bounding

- **JSONB Auto-Detection**: Automatically utilizes SQLite's binary `jsonb` functions when available for a **2-5x performance boost**.
- **Rigorous Memory Bounding**: Every internal engine now strictly uses `fetchmany(101)`, ensuring a constant memory footprint regardless of result set size.

## File Storage: Modern GridFS (`GridFSBucket`)

Completed the transition to the modern PyMongo file storage API:
- **`GridFSBucket`**: Full implementation of the modern upload/download stream API.
- **Compatibility**: Maintains 100% parity with both modern and legacy GridFS specs.
