# NeoSQLite v1.13.4 Release Notes

## TL;DR

NeoSQLite v1.13.4 is a **bug fix release** that adds **GridFS support** to NX-27017, along with performance improvements, configurable journal modes, and stability enhancements.

- **Upgrade if you want:** GridFS file storage via MongoDB wire protocol, better performance, and configurable journal modes
- **No need to upgrade if:** You're satisfied with the current NeoSQLite API
- **Breaking changes:** None

---

## Breaking Changes

**None** — v1.13.4 is fully backward compatible with v1.13.0.

---

## Overview

This release brings significant improvements to NX-27017, including:

- **GridFS Support**: Upload, download, list, and delete files via the MongoDB wire protocol
- **Configurable Journal Modes**: Choose WAL, DELETE, TRUNCATE, PERSIST, MEMORY, or OFF via `-j` CLI flag
- **Performance Improvements**: Reduced O(n²) bytes concatenation and optimized I/O loops
- **Stability Fixes**: Real uptime/memory tracking, proper connection counting, and GridFS routing fixes

---

## New Features

### NX-27017: GridFS File Storage Support

NX-27017 now supports GridFS operations through the MongoDB wire protocol, enabling PyMongo GridFS clients to store and retrieve files:

```python
from pymongo import MongoClient
from gridfs import GridFS

client = MongoClient('mongodb://localhost:27017/')
db = client.my_database
fs = GridFS(db)

# Upload a file
file_id = fs.put(b"Hello GridFS!", filename="hello.txt")

# Download a file
content = fs.get(file_id).read()

# List files
for f in fs.find():
    print(f.filename, f.length)

# Delete a file
fs.delete(file_id)
```

**Supported GridFS Operations**:

| Operation | Description |
|-----------|-------------|
| `find` | List files in `.files` collection |
| `delete` | Delete files by `_id` |
| `upload` | Upload files via `open_upload_stream` |
| `openDownloadStream` | Download files via `open_download_stream` |

### Configurable SQLite Journal Mode

NX-27017 now supports configurable SQLite journal modes via `-j` or `--journal-mode` CLI flag:

```bash
# WAL mode (default) - best concurrency
nx-27017 --db ./myapp.db

# Specify journal mode
nx-27017 --db ./myapp.db -j DELETE
nx-27017 --db ./myapp.db --journal-mode TRUNCATE
```

**Available Modes**:

| Mode | Description |
|------|-------------|
| `WAL` | Write-Ahead Logging (default) - best concurrency |
| `DELETE` | Traditional rollback journal |
| `TRUNCATE` | Truncate journal to zero length |
| `PERSIST` | Persist journal on disk |
| `MEMORY` | Journal in RAM (fast, no crash recovery) |
| `OFF` | Disable journaling |

### Real Server Statistics

NX-27017 now returns real runtime data instead of mock values:

- **Uptime**: Actual server running time (`uptime`, `uptimeMillis`, `uptimeEstimate`)
- **Memory**: Actual memory usage via `resource.getrusage()` (`mem.resident`, `mem.virtual`)
- **Connections**: Active connection counter that increments/decrements on client connect/disconnect
- **Database Size**: Real database file sizes in `listDatabases` (`sizeOnDisk`, `totalSize`)

### Index Management via Wire Protocol

NX-27017 now supports full index management commands:

| Command | Description |
|---------|-------------|
| `createIndexes` | Create multiple indexes with options |
| `createIndex` | Create single index (legacy singular form) |
| `dropIndexes` | Drop all or specific indexes |
| `dropIndex` | Drop single index (legacy singular form) |

### Aggregate Handler Simplification

Removed special-case handling for `$count` and `$match+$group` patterns. NeoSQLite's `aggregate()` now supports the full pipeline including: `$match`, `$group`, `$project`, `$sort`, `$skip`, `$limit`, `$unwind`, `$lookup`, `$facet`, `$bucket`, `$count`, `$addFields`, `$setWindowFields`, `$graphLookup`, `$fill`, `$unset`, `$replaceRoot`, `$replaceWith`, `$sample`, `$redact`, `$unionWith`, `$merge`, `$densify`.

## Bug Fixes

### Fixed: GridFS Collection Routing for .chunks

**Problem**: GridFS operations via PyMongo failed with "GridFS find only supported on .files collections" because `.chunks` collections were incorrectly routed to the GridFS handler.

**Solution**: Updated `_is_gridfs_collection` to only return `True` for `.files` collections, allowing `.chunks` to be handled as regular collections:

```python
# Before (incorrectly handled .chunks as GridFS)
def _is_gridfs_collection(self, coll_name: str) -> bool:
    return coll_name.endswith(".files") or coll_name.endswith(".chunks")

# After (only .files is GridFS)
def _is_gridfs_collection(self, coll_name: str) -> bool:
    return coll_name.endswith(".files")
```

### Fixed: GridFS Table Name Quoting

**Problem**: Table names containing dots (like `fs.files`, `fs.chunks`) were not properly quoted in SQL statements, causing SQLite to interpret them incorrectly.

**Solution**: Added `quote_table_name` utility to properly quote table names in all SQL statements:

```python
# Before (incorrect table reference)
CREATE TABLE IF NOT EXISTS fs.files (...)

# After (properly quoted)
CREATE TABLE IF NOT EXISTS "fs.files" (...)
```

### Fixed: Legacy Table Migration

**Problem**: When migrating legacy tables, if the `fs.files` rename succeeded but `fs.chunks` rename failed, both collection names were incorrectly reverted.

**Solution**: Fixed to only revert on failure of `fs.files` rename, and silently ignore `fs.chunks` rename failures:

```python
# Now handles each table independently
if cursor.fetchone():
    try:
        self._db.execute(f'ALTER TABLE "{old_files}" RENAME TO {new_files}')
    except Exception:
        self._files_collection = old_files
        self._chunks_collection = old_chunks
        return
```

### Fixed: O(n²) Bytes Concatenation in OP_MSG

**Problem**: `build_reply()` used repeated `+=` on bytes objects, causing O(n²) time complexity when building large messages.

**Solution**: Use list + join pattern for efficient concatenation:

```python
# Before (O(n²))
sections = b""
for doc in documents:
    sections += struct.pack("<B", 0) + doc_data

# After (O(n))
sections = []
for doc in documents:
    sections.append(struct.pack("<B", 0))
    sections.append(doc_data)
body = struct.pack("<I", flags) + b"".join(sections)
```

### Fixed: I/O Loop Performance

**Problem**: Reading message headers and bodies used inefficient bytes concatenation in loops.

**Solution**: Use pre-allocated `bytearray` for I/O loops:

```python
# Before (repeated concatenation)
header = b""
while len(header) < 16:
    chunk = await reader.read(16 - len(header))
    header += chunk

# After (pre-allocated buffer)
header_bytes = bytearray(16)
pos = 0
while pos < 16:
    chunk = await reader.read(16 - pos)
    header_bytes[pos : pos + len(chunk)] = chunk
    pos += len(chunk)
```

### Fixed: $count Aggregation Performance

**Problem**: `$count` aggregation stage loaded all documents into memory via `len(list(find()))`.

**Solution**: Removed special-case handling - NeoSQLite's aggregate pipeline handles $count efficiently:

```python
# Now handled by full pipeline support
cursor = coll.aggregate(pipeline)
docs = list(cursor)
```

### Fixed: Index Name Parsing

**Problem**: Index name parsing used string slicing which could be error-prone.

**Solution**: Use `removeprefix()` for cleaner extraction:

```python
# Before
key_str = idx_name[len(f"idx_{coll.name}_") :]

# After
key_str = idx_name.removeprefix(prefix).replace("_", ".")
```

### Fixed: Poetry Extras Version Constraints

**Problem**: Poetry extras had version constraints that caused dependency resolution issues.

**Solution**: Removed version constraints from nx27017 extras in pyproject.toml.

## Performance Improvements

- **I/O Reading**: Pre-allocated `bytearray` buffers avoid repeated allocation
- **Aggregate Handler**: Removed $count special case, uses full pipeline support
- **Signal Handlers**: Named function `_signal_exit_handler` replaces lambda

GridFS is now available through the wire protocol:

## Known Limitations

### NX-27017 Limitations

- **No replication/sharding**: Coming never™
- **No transactions**: Requires replica set (SQLite doesn't support MVCC)
- **Limited batch_size**: `find_raw_batches` with batch_size not implemented (requires cursor state management)
- **Change streams via replica set**: Not supported (SQLite-trigger-based `watch()` is available)

### NeoSQLite Core Limitations

- **No true transactions**: SQLite's transaction model differs from MongoDB's
- **Limited indexing**: SQLite FTS5 has different capabilities than MongoDB text indexes
- **No aggregation framework parity**: Some advanced aggregation stages not implemented
