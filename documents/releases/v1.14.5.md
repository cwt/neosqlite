# NeoSQLite v1.14.5 Release Notes

## TL;DR

NeoSQLite v1.14.5 is a **feature enhancement release** that adds native FTS5 BM25 text relevance scoring to aggregation pipelines and enables `$push`/`$addToSet` with nested object expressions in `$group` stages.

**Key Features:** This release implements `$meta: "textScore"` for native text search relevance scoring using SQLite's BM25 algorithm, and adds full support for `$push` and `$addToSet` with expression objects to collect nested structures during grouping operations.

- **Upgrade if you use:** Text search with relevance scoring, or `$group` stages that collect nested objects.
- **Result Correctness:** Achieved **100% compatibility** in API comparison tests with comprehensive test coverage for all new features.

## High Severity Enhancements

### Added $meta: textScore for Native FTS5 Relevance Scoring

**Feature:** Native FTS5 BM25 text relevance scoring is now available in aggregation pipelines via `{$meta: "textScore"}`.

**How It Works:** During `$text` search stages, NeoSQLite captures the FTS5 BM25 score and stores it in the document's JSON data. When `$meta: "textScore"` is used in `$project` or `$addFields`, the score is extracted and made available for sorting and filtering.

**SQL Implementation:**

1. **Score Capture** ‚Äî During `$text` match, `-bm25(fts_table)` is computed and stored as `_textScore` in the document JSON
2. **Score Extraction** ‚Äî `$meta: "textScore"` in `$project`/`$addFields` extracts the stored score via `json_extract(data, '$._textScore')`
3. **Python Fallback** ‚Äî When force fallback is enabled, the Python evaluator returns the stored score (or 0.0 if no FTS was used)

**Impact:**
- **Before:** No native text scoring in aggregation ‚Äî required Python post-processing
- **After:** Native BM25 scoring available directly in aggregation pipelines

**Example:**

```python
# Create FTS index
db.posts.create_index('title', fts=True)

# Use native text scoring in aggregation
pipeline = [
    {'$match': {'$text': {'$search': 'Python'}}},
    {'$project': {
        'title': 1,
        'score': {'$meta': 'textScore'}
    }},
    {'$sort': {'score': -1}}
]

results = list(db.posts.aggregate(pipeline))
# Results sorted by relevance score!
for post in results:
    print(f"Title: {post['title']}, Score: {post['score']}")
```

**With $addFields:**

```python
pipeline = [
    {'$match': {'$text': {'$search': 'Learning'}}},
    {'$addFields': {'relevance': {'$meta': 'textScore'}}},
]

results = list(db.articles.aggregate(pipeline))
# All documents include relevance score
```

**Kill Switch Support:**

```python
from neosqlite.collection.query_helper import set_force_fallback

# Force Python fallback for debugging
set_force_fallback(True)
# $meta: textScore returns 0.0 (no FTS5 in Python path)
set_force_fallback(False)  # Reset to native SQL path
```

## Medium Severity Enhancements

### Added $push and $addToSet with Expression Objects

**Feature:** `$push` and `$addToSet` now support expression objects to collect nested documents during `$group` stages.

**Problem:** Previously, `$push` with expressions like `{'$push': {'title': '$title', 'author': '$author'}}` would raise `NotImplementedError` and fall back to Python evaluation.

**Solution:** Implemented native SQL support that builds `json_object()` expressions for nested document collection:

1. **Expression Detection** ‚Äî Detects dict expressions in `$push`/`$addToSet`
2. **JSON Object Construction** ‚Äî Builds `json_object()` with field references and literal values
3. **Array Aggregation** ‚Äî Uses `json_group_array()` to collect objects into arrays

**SQL Logic:**

```sql
-- For {$push: {title: '$title', author: '$author'}}
json_group_array(json_object(
    'title', json_extract(data, '$.title'),
    'author', json_extract(data, '$.author')
)) AS items
```

**Impact:**
- **Before:** `$push` with expressions ‚Üí Python fallback
- **After:** `$push` with expressions ‚Üí Native SQL collection

**Example:**

```python
# Collect nested objects during grouping
pipeline = [
    {'$group': {
        '_id': '$category',
        'items': {
            '$push': {
                'title': '$title',
                'author': '$author'
            }
        }
    }},
    {'$sort': {'_id': 1}}
]

results = list(db.posts.aggregate(pipeline))
# Returns: {'_id': 'tech', 'items': [
#   {'title': 'Post 1', 'author': 'Alice'},
#   {'title': 'Post 2', 'author': 'Bob'}
# ]}
```

**With Literal Values:**

```python
pipeline = [
    {'$group': {
        '_id': '$category',
        'items': {
            '$push': {
                'name': '$name',
                'type': 'product',  # Literal string
                'quantity': 10       # Literal number
            }
        }
    }}
]
```

**$addToSet with Deduplication:**

```python
pipeline = [
    {'$group': {
        '_id': '$category',
        'unique_items': {
            '$addToSet': {
                'title': '$title',
                'author': '$author'
            }
        }
    }}
]
# Automatically deduplicates identical objects
```

## Migration Guide

### Upgrading from v1.14.4

This is a non-breaking feature release. All existing code continues to work without modification.

### For Users Who Need Text Relevance Scoring

If you were using Python-based text relevance scoring:

```python
# Before v1.14.5: Manual Python scoring
pipeline = [
    {'$match': {'$text': {'$search': 'Python'}}},
]

results = list(db.posts.aggregate(pipeline))
# Python post-processing required
scored_results = []
for post in results:
    score = calculate_relevance(post, 'Python')
    scored_results.append((post, score))
scored_results.sort(key=lambda x: x[1], reverse=True)

# After v1.14.5: Native SQL scoring
pipeline = [
    {'$match': {'$text': {'$search': 'Python'}}},
    {'$project': {
        'title': 1,
        'score': {'$meta': 'textScore'}
    }},
    {'$sort': {'score': -1}}
]

results = list(db.posts.aggregate(pipeline))
# Already scored and sorted by relevance!
```

### For Users Who Need Nested Object Collection

If you were using Python post-processing to build nested structures:

```python
# Before v1.14.5: Manual Python grouping
pipeline = [
    {'$match': {'status': 'published'}},
]

results = list(db.posts.aggregate(pipeline))
# Python post-processing to group by category
from collections import defaultdict
grouped = defaultdict(list)
for post in results:
    grouped[post['category']].append({
        'title': post['title'],
        'author': post['author']
    })

# After v1.14.5: Native SQL grouping
pipeline = [
    {'$match': {'status': 'published'}},
    {'$group': {
        '_id': '$category',
        'items': {
            '$push': {
                'title': '$title',
                'author': '$author'
            }
        }
    }}
]

results = list(db.posts.aggregate(pipeline))
# Already grouped with nested objects!
```
