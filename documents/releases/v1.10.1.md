# NeoSQLite v1.10.1 Release Notes

## TL;DR

NeoSQLite v1.10.1 is a maintenance release that explicitly clarifies support for Change Streams and multi-document transactions in documentation and benchmark reports.

- **Upgrade if you need:** Improved documentation and benchmark clarity regarding `watch()` and transactions.
- **No need to upgrade if:** You're on v1.10.0 and already understand that `watch()` and transactions are fully supported.
- **Breaking changes:** None — 100% backward compatible.

---

## Overview

Following feedback that some automated tools (like NotebookLM) were incorrectly summarizing NeoSQLite as lacking `watch()` support, this release focuses on making existing features more prominent and ensuring benchmark reports clearly explain why certain features are skipped during MongoDB comparisons.

## Major Feature Clarifications

### 1. Explicit `watch()` Support

While `watch()` has been supported since v0.3.0, it is now:
- Listed as a top-level feature in the `README.md`.
- Explicitly called out in the "Features" section.
- Prominently highlighted in the "Skipped Tests Note" to confirm it is **fully implemented** via SQLite triggers.

### 2. Explicit Transaction Support

Confirmed that `ClientSession` multi-document transactions are **fully supported** via SQLite SAVEPOINTs. Like `watch()`, this is now more prominently documented to avoid any misunderstanding.

## Benchmark Infrastructure Improvements

### Enhanced Report Notes

The benchmark report generator (`examples/api_comparison/reporter.py`) has been updated to include prominent notes for Change Streams and Transactions.

When these categories are skipped (because MongoDB requires a replica set for comparison), the generated Markdown report now includes:
- **Change Streams**: MongoDB skipped - MongoDB requires replica set (Fully implemented in NeoSQLite via SQLite triggers)
- **session_methods**: MongoDB skipped - MongoDB requires replica set (Fully implemented in NeoSQLite via SQLite SAVEPOINTs)

This prevents users from misinterpreting `N/A` or `0.00ms` values as a lack of feature support.
