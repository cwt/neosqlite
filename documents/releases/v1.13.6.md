# NeoSQLite v1.13.6 Release Notes

## TL;DR

NeoSQLite v1.13.6 is a **bug fix release** that improves MongoDB compatibility for string operators and SQL projections.

- **Upgrade if you use:** `$regexFind`, `$regexFindAll`, `$indexOfCP`, or field names that are SQLite reserved keywords
- **No need to upgrade if:** You don't use these operators or project with reserved keyword field names
- **Breaking changes:** None

---

## Breaking Changes

**None** — v1.13.6 is fully backward compatible with v1.13.5.

---

## Overview

This release fixes critical MongoDB compatibility issues:

- **`$regexFind` and `$regexFindAll`**: Fixed output format to match MongoDB's `idx` field name and `captures` array
- **`$indexOfCP` and `$indexOfBytes`**: Fixed swapped parameter order in SQL and Python evaluators
- **SQL Projections**: Added proper identifier quoting to prevent syntax errors with reserved keywords like `index`

---

## Bug Fixes

### Fixed: $regexFind and $regexFindAll Output Format

**Problem**: Output used `index` field name instead of MongoDB's `idx`, and was missing the `captures` array.

**Before (incorrect)**:
```python
{"match": "Alice", "index": 2}
```

**After (MongoDB-compatible)**:
```python
{"match": "Alice", "idx": 2, "captures": []}
```

**Capture Groups**: Now properly extracts capture groups:
```python
{"match": "hello world", "idx": 0, "captures": ["hello", "world"]}
```

### Fixed: $indexOfCP and $indexOfBytes Parameter Order

**Problem**: Parameters were swapped in both SQL and Python evaluators, causing incorrect results.

**Before (incorrect)**:
```python
# MongoDB: $indexOfCP expects [string, substring]
# NeoSQLite was using: instr(substr, string)  # WRONG
```

**After (correct)**:
```python
# instr(haystack, needle) - now correctly ordered
instr(string_sql, substr_sql) - 1
```

### Fixed: SQL Column Alias Quoting

**Problem**: Column aliases in SQL projections weren't quoted, causing syntax errors with SQLite reserved keywords.

**Before (could fail)**:
```sql
SELECT json_extract(data, '$.index') AS index FROM ...
-- Error: near "index": syntax error
```

**After (quoted)**:
```sql
SELECT json_extract(data, '$.index') AS "index" FROM ...
-- Works correctly
```

## NX-27017 v0.5.0

### New: GridFS Adapter

NX-27017 now includes a full GridFS adapter for MongoDB GridFS compatibility:

- Upload files via PyMongo GridFS API
- Download files with streaming support
- Query files collection with full MongoDB compatibility
- Delete files and their chunks

### Bug Fixes

- **Aggregation cursor**: Use `cursor.to_list()` instead of `list(cursor)` for proper result handling
- **Session ID extraction**: Fixed Binary format handling in session IDs
- **Transaction handling**: Fixed session tracking for commit/abort operations

## Migration Guide

### Upgrading from v1.13.x

**No breaking changes** — v1.13.6 is fully backward compatible.

### If You Use $regexFind or $regexFindAll

Your code will now receive MongoDB-compatible output:

```python
# Before: {'match': 'Alice', 'index': 2}
# After:  {'match': 'Alice', 'idx': 2, 'captures': []}

# Access results correctly:
result = collection.find_one({"name": {"$regex": "Alice"}})
idx = result["match"]["idx"]  # Correct - was "index" before
```

### If You Use $indexOfCP or $indexOfBytes

Results are now correct:

```python
# Searching for "ice" in "Alice"
# Before (wrong): 0
# After (correct): 4
```
