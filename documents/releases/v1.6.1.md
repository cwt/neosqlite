# NeoSQLite v1.6.1 Release Notes

## Overview

This is a **documentation and completion release** that achieves a significant milestone:

**100% MongoDB API Coverage** - All comparable MongoDB APIs are now tested against PyMongo, with **264 tests (261 passing, 3 skipped)** and **100% compatibility** achieved.

**Key Achievement**: NeoSQLite v1.6.1 represents **complete API parity** with MongoDB for all comparable features, with comprehensive documentation and zero missing tests.

## Major Achievement: Complete API Test Coverage

### What Changed Since v1.6.0

In v1.6.0, we achieved 100% compatibility but had 4 tests skipped. In v1.6.1, we've:

1. **Enabled `$replaceOne` comparison test** - Previously skipped due to SQL tier limitation, now passes
2. **Clarified skipped tests** - Reduced from 4 to 3 by properly categorizing NeoSQLite extensions
3. **Documented all features** - Removed MISSING-APIS.md as all features are now tested
4. **Updated all documentation** - Consistent messaging across README, scripts, and docs

### Skipped Tests (By Design)

The 3 skipped tests are **architectural differences**, not missing implementations:

1. **`watch()` (Change Streams)** - ✅ **Implemented** via SQLite triggers. Skipped because MongoDB requires a replica set for comparison testing. NeoSQLite tests: `tests/test_changestream.py`

2. **`watch()` (Collection Methods)** - ✅ **Implemented** via SQLite triggers. Same as above, tested in `tests/test_changestream.py`

3. **`$log2`** - ✅ **Implemented** using SQLite's native `log2()` function. Skipped because it's a NeoSQLite extension (not in MongoDB). Raises `UserWarning` with MongoDB alternative: `{ $log: [ <number>, 2 ] }`

#### `$replaceOne` Operator Test (Now Passing)

**Before (v1.6.0):**


**After (v1.6.1):**
```python

    result = list(
        neo_collection.aggregate([
            {"$project": {
                "replaced": {

                        "input": "$str",
```

**Result**: Test now passes, removed from skipped list.

## Upgrade Notes

### From v1.6.0

**Breaking Changes:**

- **`$toBinData` removed** - This operator was never part of MongoDB's aggregation framework. It was an experimental NeoSQLite extension that converted values to Binary objects. Since it has no MongoDB equivalent, it was removed to maintain 100% API compatibility.

  **Before (v1.6.0):**
  ```python
  # NeoSQLite-only operator
  collection.aggregate([
      {"$project": {"bin": {"$toBinData": "$data"}}}
  ])
  ```

  **After (v1.6.1):**
  ```python
  # Use standard Binary() constructor instead
  from neosqlite.binary import Binary
  collection.insert_one({"data": Binary(b"binary data")})
  ```

- **`$toRegex` removed** - This operator was never part of MongoDB's aggregation framework. It was an experimental NeoSQLite extension that converted strings to regex patterns. Since it has no MongoDB equivalent, it was removed to maintain 100% API compatibility.

  **Before (v1.6.0):**
  ```python
  # NeoSQLite-only operator
  collection.aggregate([
      {"$project": {"regex": {"$toRegex": r"\\d+"}}}
  ])
  ```

  **After (v1.6.1):**
  ```python
  # Use standard Python re.compile() or pass regex pattern directly
  import re
  pattern = re.compile(r"\d+")
  # Or use $regexMatch with string pattern
  collection.aggregate([
      {"$match": {"$expr": {"$regexMatch": {"input": "$field", "regex": r"\d+"}}}}
  ])
  ```

**Rationale**: Both operators were removed to ensure NeoSQLite maintains **100% MongoDB API compatibility**. These operators were never part of MongoDB's aggregation framework and would have caused confusion for users expecting PyMongo-compatible behavior.

**Why `$log2` is Kept**: Unlike `$toBinData` and `$toRegex`, the `$log2` operator is retained because:

1. **Convenient shorthand** - `$log2` provides a commonly-used mathematical operation (base-2 logarithm) that is more concise than the MongoDB equivalent `{ $log: [ <number>, 2 ] }`
2. **Clear migration path** - Users can easily migrate to MongoDB by replacing `{ $log2: <x> }` with `{ $log: [ <x>, 2 ] }`
3. **SQLite native support** - Uses SQLite's built-in `log2()` function, providing better performance than Python fallback
4. **Explicit warning** - Raises `UserWarning` to alert users about MongoDB incompatibility, allowing informed decisions
5. **Low risk of confusion** - Logarithmic operators are well-understood mathematical functions, unlike type conversion operators which could cause subtle data type issues
