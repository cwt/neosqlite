# NeoSQLite v1.13.5 Release Notes

## TL;DR

NeoSQLite v1.13.5 is a **performance release** that introduces **O(n+m) hash join optimization** for `$lookup` aggregation, along with memory-aware adaptive query planning.

- **Upgrade if you want:** Significantly faster `$lookup` operations on large collections
- **No need to upgrade if:** You don't use `$lookup` frequently
- **Breaking changes:** None

---

## Breaking Changes

**None** — v1.13.5 is fully backward compatible with v1.13.4.

---

## Overview

This release brings major performance improvements to `$lookup` aggregation:

- **Hash Join Optimization**: `$lookup` now uses O(n+m) hash join instead of O(n×m) correlated subquery
- **Memory-Aware Planning**: Automatically selects optimal strategy based on available memory and collection size
- **Deterministic Temp Tables**: Temp table names now use sha256 hashing for better SQLite cache utilization

---

## New: Transaction Support

NeoSQLite Core and NX-27017 now support PyMongo-compatible transactions:

### NeoSQLite Core API

```python
from neosqlite import Connection

conn = Connection("myapp.db")
session = conn.start_session()

try:
    session.start_transaction()
    conn.users.insert_one({"name": "Alice"}, session=session)
    conn.orders.insert_one({"user_id": 1, "product": "Book"}, session=session)
    session.commit_transaction()
except:
    session.abort_transaction()
finally:
    session.end_session()
```

### NX-27017 Wire Protocol

PyMongo and other MongoDB clients can now use transactions via the wire protocol:

```python
from pymongo import MongoClient

client = MongoClient('mongodb://localhost:27017/')
session = client.start_session()

try:
    session.start_transaction()
    client.db.users.insert_one({"name": "Alice"})
    client.db.orders.insert_one({"user_id": 1, "product": "Book"})
    session.commit_transaction()
except:
    session.abort_transaction()
finally:
    session.end_session()
```

**Features**:
- `startTransaction` / `commitTransaction` / `abortTransaction`
- `startSession` / `endSessions`
- Session-based transaction coordination
- Thread-safe session management

## Performance Improvements

### $lookup Hash Join Optimization

**Before (O(n×m) correlated subquery)**:

```sql
SELECT main_table.*, 
  (SELECT json_group_array(related.data) 
   FROM foreign_table related 
   WHERE related.key = main_table.key) AS joined
FROM main_table
```

**After (O(n+m) hash join)**:

```sql
-- Build hash table with index
CREATE TEMP TABLE _lookup_hash_xxx AS 
  SELECT data, CAST(key AS TEXT) as _join_key 
  FROM foreign_table;
CREATE INDEX idx ON _lookup_hash_xxx(_join_key);

-- Single JOIN with pre-aggregated results
SELECT main_table.*, aggregated.results AS joined
FROM main_table
LEFT JOIN (
  SELECT _join_key, json_group_array(json(data)) as results
  FROM _lookup_hash_xxx 
  GROUP BY _join_key
) ON CAST(main_table.key AS TEXT) = _join_key
```

**Complexity Improvement**:

| Dataset | Before (Correlated) | After (Hash Join) |
|---------|---------------------|-------------------|
| 1K × 1K | 1,000,000 iterations | 2,000 operations |
| 10K × 10K | 100,000,000 iterations | 20,000 operations |

### Memory-Aware Strategy Selection

The optimizer automatically chooses between:

- **Hash Join**: When foreign collection < 30% of available memory (faster)
- **Correlated Subquery**: When foreign collection is large (memory-efficient)

```python
# Decision is made automatically based on:
# - Estimated collection size (COUNT × AVG row size)
# - Available memory (SQLite cache × 50% or system RLIMIT_AS × 30%)
```

### Deterministic Temp Table Names

Temp table names now use sha256 hashing instead of UUID:

```python
# Before (random UUID)
"_lookup_hash_a1b2c3d4"

# After (deterministic sha256)
"_lookup_hash_abc123"  # based on collection + field + pipeline
```

**Benefits**:
- SQLite temp storage can better reuse cached pages
- Easier debugging (same query = same table names)
- Improved repeated query performance

## Bug Fixes

### Fixed: $lookup with _id Field

**Problem**: `$lookup` with `localField: "_id"` failed when `_id` wasn't stored in JSON data (stored in table column instead).

**Solution**: Added `COALESCE` fallback to use `main_table.id` when `_id` extraction returns NULL:

```sql
-- Now handles both cases
CAST(COALESCE(json_extract(data, '$._id'), main_table.id) AS TEXT)
```

### Fixed: Type Annotation in _extract_field_value

**Problem**: Mypy type checker error for variable reassignment.

**Solution**: Changed type annotation to `val: Any = doc` to allow reassignment.

### Fixed: $densify Float Conversion

**Problem**: `$densify` stage could fail with integer bounds.

**Solution**: Added explicit `float()` conversion for bounds:

```python
current = float(lower_bound)  # Now always float
```

## Migration Guide

### Upgrading from v1.13.x

**No breaking changes** — v1.13.5 is fully backward compatible.

### Using $lookup

No code changes required. The optimization is automatic:

```python
# This now automatically uses hash join when beneficial
results = collection.aggregate([
    {"$lookup": {
        "from": "other_collection",
        "localField": "_id",
        "foreignField": "userId",
        "as": "joined"
    }}
])
```

## Known Limitations

### $lookup Limitations

- **Large Foreign Collections**: Falls back to correlated subquery when collection is > 30% of available memory
- **Pipeline with $lookup**: When using `pipeline` option, always uses hash join (no memory check)

### NX-27017 Limitations

- **No replication/sharding**: Coming never™
- **Change streams via replica set**: Not supported (SQLite-trigger-based `watch()` is available)
