## Overview

NeoSQLite v1.7.0 is a **major feature and security release** that dramatically expands PyMongo compatibility and introduces comprehensive security hardening. This version implements over **70+ new MongoDB-compatible APIs**, including advanced bitwise operators, positional array updates, and complex aggregation stages.

**Key Milestone**: With this release, NeoSQLite achieves **304 automated compatibility tests** (up from 264), maintaining **100% compatibility** for all comparable features. The addition of robust SQL injection protection ensures that this expanded API surface remains secure in production environments.

## Major Achievement: 70+ New PyMongo-Compatible APIs

This release closes the gap on several "Medium Priority" features, making NeoSQLite an even more powerful drop-in replacement for PyMongo in complex applications.

### 1. Bitwise Query Operators
Full support for bitwise matching in queries:
- ✅ `$bitsAllSet` / `$bitsAllClear`
- ✅ `$bitsAnySet` / `$bitsAnyClear`

### 2. Positional Array Updates
Advanced array manipulation is now possible using positional operators:
- ✅ `$` (first matching element)
- ✅ `$[]` (all elements)
- ✅ `$[<identifier>]` with `array_filters` support for complex conditional updates.

### 3. Aggregation Framework Expansion
Implemented several high-impact stages and operators:
- **Stages**: `$bucket`, `$bucketAuto`, `$unionWith`, `$merge`, `$redact`, `$densify`.
- **Date Operators**: `$dateFromString`, `$dateToString`, `$dateDiff`, `$dateTrunc`, and more.
- **Array/Set Operators**: `$sortArray`, `$firstN`, `$lastN`, `$setUnion`, `$setIntersection`, etc.
- **Utility Operators**: `$mergeObjects`, `$getField`, `$let`, `$literal`, `$rand`.

### 4. Comprehensive Cursor & Collection APIs
Enhanced cursor control to match PyMongo's rich feature set:
- ✅ `explain()`: Returns SQLite's query execution plan.
- ✅ `collation()`: Support for case-insensitive and locale-specific sorting.
- ✅ `min()` / `max()`: Index boundary control.
- ✅ `to_list()` / `clone()`: Efficient cursor management.
- ✅ `comment()`: SQL-level query tracing.

## Security: Comprehensive SQL Injection Hardening

As the API surface grew, we prioritized a "Security First" approach to SQL generation. v1.7.0 introduces a centralized SQL utility layer (`neosqlite/sql_utils.py`) that systematically hardens the engine against injection attacks:

- **Robust Quoting**: Every table name, collection name, and internal identifier is now strictly quoted using standard SQL escaping.
- **Layered Protection**: Hardening applies to the Query Engine, SQL Translation (Tier-1), and Temporary Table Aggregation (Tier-2).
- **Safe Identifiers**: Dynamic identifiers used in `$[]` array filters and aggregation variables are now sanitized at the source.

## Proof of Compatibility: The API Comparison Suite

The strongest evidence that NeoSQLite is a true PyMongo drop-in replacement lies in our **API Comparison Suite** (`examples/api_comparison/`).

Unlike traditional unit tests, this suite executes the **exact same code** against both a live MongoDB instance and NeoSQLite, then performs a deep-equality comparison of the results (including document structures, types, and counts).

- **304 Real-world Scenarios**: Every new operator and stage is verified through this rigorous comparison.
- **Result Parity**: We don't just "support" the API; we ensure the *output* is identical to MongoDB's behavior.
- **Transparency**: Users can run `python examples/api_comparison/runner.py` themselves to verify compatibility in their own environment.

## Implementation Details

### Core Improvements
- **`_internal_update` Refactor**: Improved tracking of `modified_count` to accurately report when a document was actually changed vs. just matched.
- **Identifier Sanitization**: Integrated `quote_table_name` and `quote_identifier` across all SQL-generating methods in `query_engine.py` and `sql_translator_unified.py`.
- **Positional Logic**: Implemented complex array path parsing and filtering to support `array_filters`.

### Skipped Tests Detail (Architectural Differences)
1. **`watch()`**: Requires MongoDB replica set (tested independently via SQLite triggers).
2. **`$log2`**: NeoSQLite native extension (uses SQLite `log2()`).
