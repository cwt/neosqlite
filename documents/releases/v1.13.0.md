# NeoSQLite v1.13.0 Release Notes

## TL;DR

NeoSQLite v1.13.0 is a **major feature release** that introduces **NX-27017**, a MongoDB Wire Protocol Server backed by SQLite. This allows MongoDB clients (including PyMongo) to connect directly to NeoSQLite using the standard MongoDB protocol.

- **Upgrade if you want:** MongoDB wire protocol compatibility for existing MongoDB applications
- **No need to upgrade if:** You're satisfied with the current NeoSQLite API
- **Breaking changes:** None

## Overview

NeoSQLite v1.13.0 introduces **NX-27017**, a MongoDB Wire Protocol Server that stores data in SQLite. This enables a new use case: running existing MongoDB applications against a SQLite backend without any code changes.

**Key Achievement**: NeoSQLite now supports the full MongoDB wire protocol, enabling 376 API compatibility tests to pass against both real MongoDB and NX-27017.

## New Features

### NX-27017: MongoDB Wire Protocol Server

NX-27017 is a drop-in replacement for MongoDB when used with PyMongo or any MongoDB wire protocol-compatible client.

**Quick Start**:

```bash
# Run NX-27017 server
nx-27017 --db ./myapp.db

# Connect with PyMongo (no code changes needed)
from pymongo import MongoClient
client = MongoClient('mongodb://localhost:27017/')
db = client.my_database
db.users.insert_one({"name": "Picard", "rank": "Captain"})
db.users.find()
```

**Supported Operations**:

| Category | Commands |
|----------|----------|
| **Handshake** | `ping`, `ismaster`, `hello`, `buildInfo` |
| **CRUD** | `insert`, `find`, `update`, `delete`, `replace_one` |
| **Aggregation** | `aggregate`, `count`, `distinct` with all common stages |
| **Collections** | `create`, `drop`, `renameCollection`, `listCollections`, `listCollectionNames` |
| **Indexes** | `createIndexes`, `listIndexes`, `dropIndexes`, `listSearchIndexes` |
| **Sessions** | `startSession`, `endSessions` |
| **Query Features** | `hint`, `min`, `max`, `sort`, `skip`, `limit`, `projection` |

### uvloop Support for Async Mode

NX-27017 now uses **uvloop** for improved async performance when available:

```bash
# Install with uvloop
pip install neosqlite[nx27017-speed]

# NX-27017 automatically uses uvloop when installed
```

## Bug Fixes

### Fixed: $mod Operator with String Values

**Problem**: SQLite loosely coerces string "25" to integer 25, causing `$mod` queries to incorrectly match string fields that look like numbers.

**Solution**: Added `json_type` check to ensure only actual numeric types (integer, real) are matched:

```python
# Before (incorrectly matched string fields)
collection.find({"value": {"$mod": [5, 0]}})  # Matched "25", "30", etc.

# After (only matches actual numbers)
collection.find({"value": {"$mod": [5, 0]}})  # Only matches 25, 30, etc.
```

### Fixed: dbStats with FTS5 Virtual Tables

**Problem**: `dbStats` command failed on databases with FTS5 virtual tables due to "no such column" errors.

**Solution**: Skip FTS5 virtual tables when counting objects in `dbStats`:

```python
# Before (failed with FTS5 tables)
db.command("dbStats")  # "no such column: T.title" error

# After (skips FTS5 tables)
db.command("dbStats")  # Works correctly
```

### Fixed: Internal SQLite Tables Exposed as Collections

**Problem**: `list_collection_names()` and `list_collections()` returned internal SQLite tables like `sqlite_sequence`, which should not be visible as MongoDB collections.

**Solution**: Updated queries to filter out tables starting with `sqlite_`:

```python
# Before (exposed internal tables)
db.list_collection_names()  # ['users', 'sqlite_sequence', 'sqlite_stat1']

# After (filters internal tables)
db.list_collection_names()  # ['users']
```

### Fixed: Delete Operations on Non-Existent Collections

**Problem**: Calling `delete_one()` or `delete_many()` on a non-existent collection would raise a SQLite error instead of returning `deleted_count=0`.

**Solution**: Added table existence check before delete operations:

```python
# Before (raised error)
collection.delete_many({})  # sqlite3.OperationalError: no such table

# After (returns 0 deleted)
collection.delete_many({})  # DeleteResult(deleted_count=0)
```

## Migration Guide

### Upgrading from v1.12.x

**No breaking changes** â€” v1.13.0 is fully backward compatible.

### New: Using NX-27017

To use NX-27017 as a MongoDB replacement:

1. **Install**:

   ```bash
   pip install "neosqlite[nx27017]"
   ```

2. **Start server**:

   ```bash
   nx-27017 --db ./myapp.db
   ```

3. **Connect with PyMongo**:

   ```python
   from pymongo import MongoClient
   client = MongoClient('mongodb://localhost:27017/')
   # Use as normal MongoDB
   ```

### Converting Existing MongoDB Data

To migrate from MongoDB to NeoSQLite:

1. Export data from MongoDB (e.g., using `mongodump`)
2. Start NX-27017 with a new database
3. Import data using PyMongo (no code changes needed)

## Known Limitations

### NX-27017 Limitations

- **No replication/sharding**: Coming neverâ„˘
- **No GridFS**: MongoDB-specific file storage not implemented
- **No transactions**: Requires replica set (SQLite doesn't support MVCC)
- **Limited batch_size**: `find_raw_batches` with batch_size not implemented (requires cursor state management)
- **Change streams via replica set**: Not supported (SQLite-trigger-based `watch()` is available)

### NeoSQLite Core Limitations

- **No true transactions**: SQLite's transaction model differs from MongoDB's
- **Limited indexing**: SQLite FTS5 has different capabilities than MongoDB text indexes
- **No aggregation framework parity**: Some advanced aggregation stages not implemented
