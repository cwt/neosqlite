# NeoSQLite v1.14.10 Release Notes

## TL;DR

NeoSQLite v1.14.10 is a **bug fix release** that resolves the `$in`/`$nin` operator bug for both `find()` and aggregation pipelines, adds proper MongoDB array semantics for comparison operators, and introduces dynamic `__version__` attribute.

**Key Fixes:**
1. **`$in`/`$nin` on array fields** — Both `find()` and aggregation pipelines now properly handle `$in`/`$nin` operators on array fields via Python fallback. Previously returned 0 results due to incorrect SQL translation.
2. **Array value query support** — Operators `$eq`, `$ne`, `$gt`, `$gte`, `$lt`, `$lte` now detect array query values and fall back to Python with correct MongoDB array semantics (ANY element matches).
3. **MongoDB array semantics** — Python fallback operators (`_eq`, `_ne`, `_gt`, `_lt`, `_gte`, `_lte`, `_mod`, `_nin`) now correctly handle array document fields following MongoDB semantics.
4. **Dynamic `__version__`** — `neosqlite.__version__` now returns version from `pyproject.toml` without hardcoding.

- **Upgrade if you use:** `$in`/`$nin` on array fields, or array comparison queries.
- **Result Correctness:** `$in`/`$nin` now return correct results matching MongoDB behavior — no more empty results.

---

## Overview

This release fixes bugs across two severity levels:

- **$in/$nin Array Fields Bug** — Both `find()` and aggregation pipelines returned 0 results when using `$in`/`$nin` on array fields due to incorrect SQL translation comparing JSON array strings against scalar values.
- **Array Value Queries** — Queries with array values (e.g., `{"$eq": [1, 2, 3]}`) now fall back to Python instead of crashing with SQL binding errors.
- **MongoDB Array Semantics** — Python fallback operators now follow MongoDB semantics where comparison operators match if ANY array element satisfies the condition.

## High Severity Fixes

### Fixed $in/$nin on Array Fields Returning 0 Results

**Bug:** `$in` and `$nin` on array fields (e.g., `{"tags": {"$in": ["value"]}}`) returned 0 results in both `find()` and aggregation pipelines.

**Error Symptom:**

```python
# Both returned 0 results before fix
r1 = list(collection.find({"tags": {"$in": ["maliit-keyboard"]}}))
r2 = list(collection.aggregate([
    {"$match": {"tags": {"$in": ["maliit-keyboard"]}}}
]))
```

**Root Cause:** The SQL translation of `$in`/`$nin` generates simple `IN`/`NOT IN` clauses:
```sql
json_extract(data, '$.tags') IN ('value')
```

This doesn't work for array fields because `json_extract` returns the entire JSON array as a string, and comparing this string against scalar values never matches. The proper array-aware logic exists in Python filtering (`_in()` and `_nin()` functions in `query_operators.py`) but SQL translators weren't falling back to it.

**Solution:** Both `query_builder.py` and `sql_translator_unified.py` now return `(None, [])` for `$in`/`$nin` on non-`_id` fields, triggering Python fallback which has correct array semantics:

```python
def _in(field: str, value: list[Any], document: dict[str, Any]) -> bool:
    doc_value = _get_nested_field(field, document)
    # If the field value is a list, check if any element is in the provided list
    if isinstance(doc_value, list):
        return any(item in value for item in doc_value)
    else:
        # If the field value is not a list, check if it's in the provided list
        return doc_value in value

def _nin(field: str, value: list[Any], document: dict[str, Any]) -> bool:
    doc_value = _get_nested_field(field, document)
    # If the field value is a list, check if none of the elements are in the provided list
    if isinstance(doc_value, list):
        return not any(item in value for item in doc_value)
    else:
        return doc_value not in values
```

**Impact:**
- **Before:** `$in`/`$nin` on array fields → 0 results (SQL comparison fails)
- **After:** Falls back to Python → correct array element matching

**Affected Syntax Variants:**
- `{"field": {"$in": ["value"]}}` — ✅ Now works correctly
- `{"field": {"$nin": ["value"]}}` — ✅ Now works correctly
- `$in`/`$nin` in aggregation `$match` — ✅ Now works correctly

## Medium Severity Fixes

### Fixed Array Value Queries (No More SQL Binding Errors)

**Bug:** Queries with array values (e.g., `{"field": {"$eq": [1, 2, 3]}}`) raised `sqlite3.ProgrammingError: type 'list' is not supported` when trying to bind to SQL parameters.

**Solution:** Operators `$eq`, `$ne`, `$gt`, `$gte`, `$lt`, `$lte` now detect `isinstance(value, (list, tuple))` and return `(None, [])` to trigger Python fallback:

```python
case "$eq":
    # Array values need Python for correct semantics
    if isinstance(op_val, (list, tuple)):
        return None, []
    # ... generate SQL for scalar values
```

**Impact:**
- **Before:** Array value queries → SQL binding error crash
- **After:** Array value queries → Python fallback with correct semantics

---

### Improved MongoDB Array Semantics in Python Fallback

**Bug:** Python fallback operators used simple equality/comparison which didn't match MongoDB's array semantics.

**MongoDB Semantics:**
- `$eq`: If doc field is array, check if value is IN array. If both are arrays, exact equality.
- `$ne`: If doc field is array, check if value is NOT IN array.
- `$gt`/`$lt`/`$gte`/`$lte`: If doc field is array, match if ANY element satisfies condition.
- `$mod`: If doc field is array, match if ANY element satisfies condition.
- `$nin`: If doc field is array, check if NONE of the elements are in the query list.

**Solution:** Updated all Python fallback operators to follow MongoDB semantics:

```python
def _eq(field, value, document):
    doc_value = _get_nested_field(field, document)
    # Both are arrays: exact array equality
    if isinstance(doc_value, list) and isinstance(value, list):
        return doc_value == value
    # Doc is array, value is scalar: check if value is in array
    if isinstance(doc_value, list):
        return value in doc_value
    # Scalar semantics: exact equality
    return doc_value == value

def _gt(field, value, document):
    doc_value = _get_nested_field(field, document)
    # MongoDB array semantics: match if ANY element satisfies
    if isinstance(doc_value, list):
        return any(isinstance(x, (int, float)) and x > value for x in doc_value)
    return doc_value > value
```

**Impact:**
- **Before:** Python fallback used simple Python comparison (incorrect for arrays)
- **After:** Python fallback follows MongoDB array semantics

### Added Dynamic `__version__` Attribute

**Feature:** `neosqlite.__version__` now returns the version from `pyproject.toml` without hardcoding:

```python
>>> import neosqlite
>>> neosqlite.__version__
'1.14.10'
```

Implemented using `importlib.metadata.version()` with fallback to `"dev"` for development mode.
