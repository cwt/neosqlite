# NeoSQLite v1.13.9 Release Notes

## TL;DR

NeoSQLite v1.13.9 is a **code organization and architecture release** that splits the monolithic NX-27017 module (2705 lines) into focused, maintainable components and adds full `$changeStream` support for PyMongo change streams.

- **Upgrade if you use:** NX-27017 wire protocol server, or need change stream support via PyMongo
- **No need to upgrade if:** You're using NeoSQLite's direct Python API exclusively
- **Breaking changes:** None — all existing code continues to work

---

## Breaking Changes

**None** — v1.13.9 is fully backward compatible with v1.13.8.

---

## Overview

This release focuses on code organization and expanding wire protocol support:

- **Monolithic file split**: 2705-line `nx_27017.py` split into 5 focused modules
- **$changeStream support**: PyMongo's `watch()` now works via NX-27017 wire protocol
- **Test infrastructure**: Fixed API comparison tests for change streams and bulk operations
- **Type safety**: All new modules pass strict mypy type checking

---

## Major Refactoring: NX-27017 Module Split

### Before: Single 2705-Line Monolith

```python
# nx_27017.py - 2705 lines handling everything
from nx_27017.nx_27017 import NeoSQLiteHandler, OP_MSG, run_server, daemonize
```

### After: 5 Focused Modules

```python
# wire_protocol.py - Wire protocol parsers (OP_MSG, OP_QUERY, ResponseBuilder)
from nx_27017.wire_protocol import OP_MSG, OP_QUERY, ResponseBuilder

# handler.py - MongoDB command translation (NeoSQLiteHandler)
from nx_27017.handler import NeoSQLiteHandler

# server.py - Network server implementation
from nx_27017.server import run_server, handle_client

# daemon.py - PID management and daemonization
from nx_27017.daemon import daemonize, write_pid_file

# changestream.py - Wire-protocol change stream support
from nx_27017.changestream import ChangeStreamManager

# nx_27017.py - Thin entry point (main() only)
from nx_27017.nx_27017 import main
```

### Benefits

- **Better maintainability**: Each module has a single, clear responsibility
- **Easier testing**: Modules can be tested in isolation
- **Reduced merge conflicts**: Smaller, focused files
- **Better IDE support**: Modern Python with full type annotations
- **No breaking changes**: Backward-compatible imports preserved

## New Features

### Added: $changeStream Support for PyMongo

**New**: NX-27017 now supports MongoDB change streams via the wire protocol, enabling PyMongo's `watch()` method to work with SQLite backend.

**Before (PyMongo → NX-27017)**:

```python
from pymongo import MongoClient

client = MongoClient('mongodb://localhost:27017')
collection = client.db.collection

# This would fail with "Aggregation stage '$changeStream' not supported"
change_stream = collection.watch()
```

**After (works!)**:

```python
from pymongo import MongoClient

client = MongoClient('mongodb://localhost:27017')
collection = client.db.collection

# Now works - creates change stream via wire protocol
change_stream = collection.watch()
for change in change_stream:
    print(change)
```

**How It Works**:

1. PyMongo sends `aggregate` command with `{$changeStream: {}}` stage
2. NX-27017 detects the `$changeStream` stage in the pipeline
3. Creates a `ChangeStreamCursor` with integer cursor ID
4. Returns proper MongoDB wire protocol response with resume token
5. PyMongo receives the cursor and can iterate over changes

**Implementation** (`changestream.py`):

```python
class ChangeStreamCursor:
    """Cursor-like object for change streams via wire protocol."""
    
    def __init__(self, collection_name, pipeline, ...):
        self._id = next(_cursor_id_counter)  # Integer for wire protocol
        self._stream_id = str(uuid.uuid4())  # Internal tracking
        self._changes = []
        # ...
    
    def get_resume_token(self) -> dict:
        return {"_id": ObjectId()}  # MongoDB-compatible resume token
```

## Bug Fixes

### Fixed: Null Check for benchmark_reporter

**Problem**: `bulk_executors.py` called `benchmark_reporter.mark_mongo_skipped()` without checking if `benchmark_reporter` was `None`, causing AttributeError.

**Before**:

```python
# Would crash if benchmark_reporter was None
benchmark_reporter.mark_mongo_skipped(...)
```

**After**:

```python
# Safe null check
if benchmark_reporter:
    benchmark_reporter.mark_mongo_skipped(...)
```

### Fixed: Collection Methods watch() Comparison

**Problem**: When comparing NeoSQLite `watch()` with MongoDB via API comparison tests, the results were reported as incompatible when they should both show "OK".

**Before**:

```python
# Reported different implementation strings
reporter.record_comparison(
    "Collection Methods", "watch",
    "IMPLEMENTED (SQLite triggers)",  # NeoSQLite
    "IMPLEMENTED (replica set)"       # MongoDB
)
# Result: Marked as incompatible (value mismatch)
```

**After**:

```python
# Report same value when both work
if mongo_watch:
    reporter.record_comparison(
        "Collection Methods", "watch",
        "OK", "OK",  # Both work = compatible
        skip_reason=None
    )
```

## Migration Guide

### Upgrading from v1.13.8

**No breaking changes** — v1.13.9 is fully backward compatible.

### If You Use NX-27017

Your existing code continues to work unchanged:

```bash
# Same as before
nx-27017 --db ./myapp.db
```

The internal refactoring improves maintainability but doesn't change the CLI or wire protocol behavior.

### If You Use Change Streams via NX-27017

You can now use PyMongo's `watch()` with NX-27017:

```python
from pymongo import MongoClient

client = MongoClient('mongodb://localhost:27017')
collection = client.db.collection

# This now works!
with collection.watch() as stream:
    for change in stream:
        print(change)
```

## Known Limitations

### NX-27017 Limitations

- **No replication/sharding**: Coming never™ — This is NX-class, not NCC-1701!
- **Change stream persistence**: Changes are tracked in memory; restart server = lost streams
- **`find_raw_batches` with batch_size**: Requires cursor state management
