# NeoSQLite v1.14.14 Release Notes

## TL;DR

NeoSQLite v1.14.14 is a **security hardening, code quality, and modernization release** that patches SQL injection vectors, improves transaction atomicity, and modernizes the codebase for Python 3.8+.

**Key Fixes:**
1. **SQL Injection Hardening** — Parameterized `load_extension()`, added PRAGMA whitelist validation, and applied `quote_table_name()` to all GridFS dynamic table constructions.
2. **Atomicity Fix** — Moved `ChangeStream._last_id` update inside `BEGIN IMMEDIATE` transaction to prevent race conditions.
3. **Code Quality** — Removed dead/unreachable code, replaced bare `except: pass` with logged debug output, and fixed `$currentDate` type_spec validation.
4. **Python 3.8+ Modernization** — Applied modern Python patterns (`@cache`, `:=`, `slots=True`, `collections.abc`) for cleaner and faster code.

- **Upgrade if you use:** `load_extension()` with untrusted input, GridFS with dynamic collection names, Change Streams, or PRAGMA commands.
- **Impact:** Hardens security posture and improves code maintainability. No breaking API changes.

---

## Overview

This release focuses on security hardening and code quality improvements identified during a comprehensive audit. It also modernizes the codebase to leverage Python 3.8/3.9/3.10 features now that NeoSQLite has dropped older Python support.

### Security Hardening

NeoSQLite v1.14.14 addresses several potential attack vectors:

#### SQL Injection: `load_extension()`
**Before:** The `load_extension()` method passed the extension path as a raw string literal.
```python
execute(f"SELECT load_extension('{path}')")
```
**After:** The extension path is passed as a bound parameter, preventing injection via malicious path strings.
```python
execute("SELECT load_extension(?)", (path,))
```

#### SQL Injection: GridFS Dynamic Table Names
All dynamic table name constructions in GridFS (`grid_file.py`, `gridfs_bucket.py`, `utils.py`) now use `quote_table_name()` to prevent identifier injection.

#### PRAGMA Validation
- **`wal_checkpoint`**: Only accepts whitelisted modes — `PASSIVE`, `FULL`, `RESTART`, `TRUNCATE`.
- **Fallback PRAGMA commands**: Validated against a regex pattern to reject unexpected command names.

### Atomicity Fix: Change Streams

The `_last_id` update for Change Streams was previously performed outside the `BEGIN IMMEDIATE` transaction block. In rare concurrent scenarios, this could cause the resume token to incorrectly advance before the corresponding change was committed. It is now moved inside the transaction for proper atomicity.

### Code Quality Cleanup

| Issue | Files |
|-------|-------|
| `except Exception: pass` → logged debug output | `temporary_table_aggregation.py` |
| `is True` / `is False` → `isinstance()` / truthiness | `query_builder.py`, `update_operations.py` |
| `hasattr` guards → `getattr(..., default)` | `connection.py` |
| Removed unreachable dead code | `cursor.py`, `query_operators.py` |
| Fixed `$currentDate` type_spec to check `"$type"` field | `update_operations.py` |

### Python 3.8+ Modernization

| Change | Impact |
|--------|--------|
| `@lru_cache(maxsize=1000)` → `@cache` | `text_search.py` — simpler, faster cached FTS queries |
| `typing.*` → `collections.abc` imports | `Callable`, `Iterable`, `Iterator` uses standardized location |
| `slots=True` on `BulkOperation` dataclasses | Reduced memory overhead for bulk operations |
| Walrus operator (`:=`) | Cleaner ID lookups in `GridFSBucket` |
| Removed unreachable dead code | `delete_many` fallback logic simplified |

## Maintenance Improvements

### Testing & Benchmark Infrastructure

- **Shared PyMongo Connection**: All API comparison tests now use a single shared `MongoClient` managed by `runner.py`, eliminating unfair benchmark overhead from repeated connection setup/teardown.
- **macOS ARM Native MongoDB**: Benchmark scripts now prefer native Homebrew `mongodb-community` on macOS before falling back to containers.
- **macOS Symlink Fix**: Fixed test failure on macOS caused by symlinked temp directories.
- **Removed Hardcoded Path**: `run-api-nx-27017.sh` no longer uses a hardcoded binary path.

### Dependencies

- Updated all locked dependencies in `poetry.lock` and `packages/nx_27017/poetry.lock`.

## Migration Guide

### Upgrading from v1.14.13

This is a non-breaking release. All existing code continues to work without modification. The security hardening changes are internal and do not affect the public API.

**Recommended for all users**, especially those running:
- Production deployments where SQL injection hardening matters
- Applications using GridFS with user-provided collection names
- Systems leveraging Change Streams where atomicity is critical
