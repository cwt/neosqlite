# NeoSQLite v1.14.6 Release Notes

## TL;DR

NeoSQLite v1.14.6 is a **bug fix release** that resolves a critical `$facet` aggregation stage failure in the Tier 2 (temporary table) processing path.

**Key Fix:** `$facet` stage now properly executes sub-pipelines using Tier 1/2/3 optimization instead of raising `NotImplementedError`. Each sub-pipeline runs independently on its best tier, then results are merged into a single document.

- **Upgrade if you use:** `$facet` aggregation stages with complex pipelines that hit Tier 2 processing.
- **Result Correctness:** Achieved **100% compatibility** with comprehensive test coverage for all `$facet` scenarios.

---

## Overview

This release focuses on aggregation pipeline reliability:

- **$facet Tier 2 Support** — Fixed missing case handler that caused `NotImplementedError` when `$facet` pipelines required temporary table processing.
- **Proper _id Preservation** — `$facet` sub-pipelines correctly preserve original document `_id` values from temporary tables.
- **Empty Collection Handling** — `$facet` with empty input collections now matches Tier 3 Python fallback behavior.
- **Comprehensive Test Coverage** — Added 3 new unit tests plus all existing `$facet` tests now pass.

## High Severity Fixes

### Fixed $facet Stage Missing Implementation in Tier 2

**Bug:** `$facet` stage was listed in `supported_stages` but had no case handler in `process_pipeline()`, causing `NotImplementedError` when pipelines required temporary table processing.

**Error Message:**

```text
NotImplementedError: Stage '$facet' not yet supported in temporary table approach
```

**Root Cause:** The `supported_stages` set included `$facet`, but the match-case block in `process_pipeline()` had no handler for it, falling through to the default case which raised the error.

**Solution:** Implemented `_process_facet_stage()` method that follows the split-execute-merge pattern:

1. **Extract Input Documents** — Retrieves documents from current temporary table, preserving `_id` from the separate column
2. **Split Sub-Pipelines** — For each facet field, executes its sub-pipeline independently
3. **Execute on Best Tier** — Each sub-pipeline uses normal aggregation (Tier 1 SQL → Tier 2 Temp Tables → Tier 3 Python)
4. **Merge Results** — Combines all facet results into a single document and stores in a new temporary table

**Impact:**
- **Before:** `$facet` with complex pipelines → `NotImplementedError` → forced Python fallback
- **After:** `$facet` with complex pipelines → Each sub-pipeline optimized independently → Results merged

**Example:**

```python
# This now works correctly with Tier 2 optimization
pipeline = [
    {
        "$facet": {
            "posts": [
                {"$unwind": "$tags"},
                {"$group": {"_id": "$tags", "count": {"$sum": 1}}}
            ],
            "total": [{"$count": "count"}],
            "categories": [
                {"$group": {"_id": "$category", "avg_score": {"$avg": "$score"}}}
            ]
        }
    }
]

results = list(db.posts.aggregate(pipeline))
# results[0]["posts"] - tag counts
# results[0]["total"] - total count
# results[0]["categories"] - category averages
```

**_id Preservation:**

```python
# Documents maintain their original _id values through $facet
collection.insert_many([
    {"_id": 1, "name": "Alice"},
    {"_id": 2, "name": "Bob"}
])

pipeline = [{"$facet": {"users": [{"$sort": {"name": 1}}]}}]
results = list(collection.aggregate(pipeline))
# results[0]["users"][0]["_id"] == 1  ✓ Preserved
# results[0]["users"][1]["_id"] == 2  ✓ Preserved
```

**Empty Collection Handling:**

```python
# Empty collection - matches Tier 3 behavior
pipeline = [{"$facet": {"results": [{"$match": {"x": 1}}], "count": [{"$count": "total"}]}}]
results = list(empty_collection.aggregate(pipeline))
# results[0]["results"] == []
# results[0]["count"] == []  # Matches Tier 3, not [{'total': 0}]
```

## Medium Severity Improvements

### Proper _id Column Extraction from Temporary Tables

**Problem:** Documents extracted from temporary tables lost their `_id` values because `_id` is stored as a separate SQLite column, not in the JSON `data` column.

**Solution:** Modified document extraction to `SELECT _id, data` and merge the `_id` into the extracted document before processing.

**SQL Implementation:**

```python
# Before: Lost _id
cursor = db.execute("SELECT data FROM temp_table")
docs = [json_loads(row[0]) for row in cursor.fetchall()]

# After: Preserves _id
cursor = db.execute("SELECT _id, data FROM temp_table")
docs = []
for row_id, row_data in cursor.fetchall():
    doc = json_loads(row_data)
    doc["_id"] = row_id
    docs.append(doc)
```

**Impact:** Ensures document identity is preserved through all temporary table operations, critical for `$facet` and other stages that re-insert documents.

## Low Severity Improvements

### Test Coverage Enhancements

## Migration Guide

### Upgrading from v1.14.5

This is a non-breaking bug fix release. All existing code continues to work without modification.

### For Users Experiencing $facet NotImplementedError

If you were encountering this error:

```text
NotImplementedError: Stage '$facet' not yet supported in temporary table approach
```

**After v1.14.6:** The error is resolved. `$facet` pipelines that previously forced Tier 3 Python fallback will now use Tier 2 temporary table optimization when appropriate:

```python
# This now works without errors
pipeline = [
    {"$match": {"status": "published"}},
    {
        "$facet": {
            "by_category": [
                {"$group": {"_id": "$category", "count": {"$sum": 1}}}
            ],
            "recent": [
                {"$sort": {"date": -1}},
                {"$limit": 10}
            ]
        }
    }
]

results = list(db.posts.aggregate(pipeline))
# No more NotImplementedError!
```
