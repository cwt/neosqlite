# NeoSQLite v1.14.12 Release Notes

## TL;DR

NeoSQLite v1.14.12 is a **bug fix release** that corrects the CTE-based array operators implementation introduced in v1.14.11, fixing a "malformed JSON" error when using `$in`, `$nin`, or `$all` operators.

**Key Fix:**
1. **Fixed malformed JSON error** — The `json_each()` function requires `json_each(data, '$.field')` syntax, not `json_each(json_extract(data, '$.field'))`. This bug caused `sqlite3.OperationalError: malformed JSON` in aggregation pipelines.

- **Upgrade if you use:** `$in`, `$nin`, or `$all` operators on array fields in aggregation pipelines.
- **Impact:** These operators now work correctly instead of raising "malformed JSON" errors.

---

## Overview

This release fixes a critical bug introduced in v1.14.11 where the CTE-based array operator optimization caused SQL errors.

### The Bug

When implementing the CTE optimization for `$in`, `$nin`, `$all` operators in v1.14.11, the code incorrectly passed a `json_extract()` expression to `json_each()`:

**Incorrect (v1.14.11):**
```sql
-- field_access = "json_extract(data, '$.field')"
EXISTS (SELECT 1 FROM json_each(json_extract(data, '$.field')) WHERE ...)
```

This produces `malformed JSON` because `json_each()` expects a JSON array as its first argument, not a scalar value extracted from one.

**Correct:**
```sql
-- json_path = "$.field"
EXISTS (SELECT 1 FROM json_each(data, '$.field') WHERE ...)
```

The `json_each(data, '$.field')` syntax correctly extracts the field from the `data` column and iterates over it as an array.

---

## Bug Details

### Affected Code

Two files had the incorrect implementation:

1. **`sql_tier_aggregator.py`** - For aggregation `$match` stage
2. **`sql_translator_unified.py`** - For `find()` queries

### Root Cause

The `SQLOperatorTranslator.translate_operator()` method receives `field_access` which is a **full expression** like `json_extract(data, '$.field')`, not the raw field name. The code incorrectly passed this expression to `json_each()`.

### Solution

1. **In `sql_tier_aggregator.py`**: Fixed by extracting the JSON path properly using `parse_json_path(field)` and constructing `json_each(data, '$.field')` with the correct syntax.

2. **In `sql_translator_unified.py`**: Reverted to Python fallback for non-`_id` fields since the translator only has access to the computed expression, not the raw field name and path separately.

## Migration Guide

### Upgrading from v1.14.11

This is a non-breaking bug fix release. All existing code continues to work without modification.

### If You Were Affected by the Bug

**Issue: `sqlite3.OperationalError: malformed JSON` when using `$in`, `$nin`, or `$all` on array fields**

```python
# Before v1.14.12: Raised "malformed JSON" error
results = list(collection.aggregate([
    {"$match": {"tags": {"$in": ["python", "nodejs"]}}}
]))

# After v1.14.12: Works correctly
results = list(collection.aggregate([
    {"$match": {"tags": {"$in": ["python", "nodejs"]}}}
]))
```

**No code changes needed** — this is a transparent fix.
