# NeoSQLite v1.12.1 Release Notes

## TL;DR

NeoSQLite v1.12.1 is a **bug fix release** that addresses PyMongo 4.16+ compatibility for `Cursor.min()`, `Cursor.max()`, and `Cursor.hint()` methods.

- **Upgrade if you use:** PyMongo 4.16+ with cursor min/max/hint methods.
- **No need to upgrade if:** You're using older PyMongo versions or don't use cursor bounds.
- **Breaking changes:** Yes — `cursor.min()` and `cursor.max()` now require list/tuple format instead of dict.

---

## Breaking Changes

### cursor.min() and cursor.max() Require List Format

**Before (v1.12.0)**: These methods accepted dict format:
```python
cursor.min({"field": 10})  # Dict format - NOW REJECTED
```

**After (v1.12.1)**: These methods require list format:
```python
cursor.hint([("field", 1)]).min([("field", 10)])  # List format - REQUIRED
```

**Migration Required**: Update all calls to `cursor.min()` and `cursor.max()` to use list format with `hint()`.

---

## Overview

NeoSQLite v1.12.1 fixes a compatibility issue with PyMongo 4.16+, which introduced stricter type validation for cursor bound methods. This release ensures NeoSQLite behaves consistently with MongoDB's wire protocol expectations.

**Key Achievement**: This release fixes cursor bound methods to match PyMongo 4.16+ behavior, but introduces a **breaking change** for code using dict format.

## Bug Fixes

### Fixed: Cursor.min() and Cursor.max() Type Validation

**Problem**: PyMongo 4.16+ enforces that `cursor.min()` and `cursor.max()` must receive a list of tuples (e.g., `[("field", value)]`) instead of a dictionary. NeoSQLite was accepting dict format, which would fail when used with real MongoDB wire protocol clients.

**Solution**: Updated `Cursor.min()` and `Cursor.max()` to:

1. Accept list/tuple format matching PyMongo API: `[("field", value)]`
2. Convert to internal dict format: `{"field": value}`
3. Raise `TypeError` if dict format is passed (matching PyMongo 4.16+ behavior)

**Before (would fail with PyMongo 4.16+)**:
```python
# This worked with NeoSQLite but failed with real MongoDB
cursor = collection.find({}).min({"field": 10})
```

**After (works with both)**:
```python
# Correct format for PyMongo 4.16+
cursor = collection.find({}).hint([("field", 1)]).min([("field", 10)])
```

### Fixed: Cursor.hint() Accepts List Format

**Problem**: PyMongo sends hints as a list of tuples when using cursor methods, but NeoSQLite only accepted string index names.

**Solution**: Updated `Cursor.hint()` to accept both:

1. String format (index name): `"field_1"`
2. List format (field, direction): `[("field", 1)]`

**Usage**:
```python
# String format
cursor = collection.find({}).hint("field_1")

# List format (PyMongo 4.16+ compatible)
cursor = collection.find({}).hint([("field", 1)])
```

## PyMongo Version Compatibility

| PyMongo Version | min/max/hint Behavior |
|-----------------|------------------------|
| PyMongo 3.x-4.15 | Accept dict, convert to list internally |
| **PyMongo 4.16+** | **Requires list/tuple, rejects dict** |

NeoSQLite v1.12.1 now matches PyMongo 4.16+ behavior across all supported PyMongo versions.

## Migration Guide

### Upgrading from v1.12.0

**⚠️ Breaking Change**: `cursor.min()` and `cursor.max()` now reject dict format. This requires code changes if you use these methods.

**If you use min/max/hint with dict format**:

```python
# Before (v1.12.0)
cursor = collection.find({}).min({"field": 10})

# After (v1.12.1) - requires hint() for proper index bounds
cursor = collection.find({}).hint([("field", 1)]).min([("field", 10)])
```

**If you use min/max with existing filter**:

```python
# Before
cursor = collection.find({"category": "A"}).min({"value": 5})

# After
cursor = collection.find({"category": "A"}).hint([("value", 1)]).min([("value", 5)])
```

**Note**: The `hint()` is required when using `min()`/`max()` to ensure the correct index is used for bounds. This matches MongoDB's behavior.

### Code Changes Required

| Scenario | Change |
|----------|--------|
| `cursor.min({...})` | **BREAKING**: Change to `cursor.hint([(...)]).min([(...)])` |
| `cursor.max({...})` | **BREAKING**: Change to `cursor.hint([(...)]).max([(...)])` |
| `cursor.hint("name")` | No change needed (still supported) |
| `cursor.hint([(...)])` | No change needed (still supported) |

## Known Limitations

### Index Requirement

For `min()` and `max()` to work correctly, an index must exist on the specified field(s). This is consistent with MongoDB's behavior, which requires an index for index bounds to function properly.

```python
# Create index first (required)
collection.create_index([("field", 1)])

# Then use min/max with hint
cursor = collection.find({}).hint([("field", 1)]).min([("field", 10)])
```
