# NeoSQLite v1.14.13 Release Notes

## TL;DR

NeoSQLite v1.14.13 is a **bug fix and maintenance release** that ensures compatibility with newer SQLite versions (3.45.0+) and restores the full index comparison suite in the API benchmarks.

**Key Fixes:**
1. **SQLite 3.45.0+ Compatibility** — Fixed a `no such column: json_each.value` error occurring on systems running newer SQLite versions by adding explicit aliases to table-valued function calls.
2. **Restored Index Benchmarks** — Re-added and enhanced the index and compound index operations comparison in the API comparison suite.

- **Upgrade if you use:** `$in`, `$nin`, `$all`, or `$elemMatch` on systems with SQLite 3.45.0+ (including many recent Linux distributions and AARCH64 environments).
- **Impact:** Prevents SQL errors during array-based queries and improves benchmark visibility.

---

## Overview

This release addresses a compatibility regression introduced by dynamic JSONB detection and restores visibility into index performance.

### The Bug: SQLite 3.45.0+ JSONB Compatibility

NeoSQLite dynamically selects `jsonb_each()` for better performance on newer SQLite versions. However, the SQL builder previously hardcoded `json_each.value` as the column qualifier. On some SQLite 3.45.0+ environments, this caused an `OperationalError: no such column: json_each.value` because the function name in the query was `jsonb_each`.

**Solution:**
By adding an explicit `AS json_each` alias to all table-valued function calls ($in, $nin, $all, $elemMatch), we ensure that column references remain valid regardless of whether the underlying function is `json_each` or `jsonb_each`.

```sql
-- Before: Could fail if json_each was replaced by jsonb_each
SELECT 1 FROM jsonb_each(data, '$.tags') WHERE json_each.value = ?

-- After: Always works
SELECT 1 FROM jsonb_each(data, '$.tags') AS json_each WHERE json_each.value = ?
```

## Maintenance Improvements

### API Comparison Suite

- **Restored Index Operations**: Re-added the index and compound index operations comparison tests.
- **Enhanced Connection Logic**: The comparison scripts now more reliably connect to PyMongo (for both real MongoDB and NX-27017 backends) across different environments.
- **Robustness**: Unit tests updated to be more version-agnostic regarding generated SQL, reducing false positives in CI environments.

## Migration Guide

### Upgrading from v1.14.12

This is a non-breaking bug fix release. All existing code continues to work without modification. It is highly recommended for users on AARCH64 or recent Linux distributions (Fedora 40+, Ubuntu 24.04+) where SQLite 3.45+ is standard.
