# NeoSQLite v1.13.7 Release Notes

## TL;DR

NeoSQLite v1.13.7 is a **bug fix release** that improves MongoDB compatibility by adding `$collStats` aggregation stage support and fixing edge cases in aggregation pipeline parsing and `collStats` command.

- **Upgrade if you use:** `$collStats` aggregation stage, `collStats` command, or aggregation pipelines with whitespace-prefixed stage names
- **No need to upgrade if:** You don't use these features and your current deployment is stable
- **Breaking changes:** None

---

## Breaking Changes

**None** — v1.13.7 is fully backward compatible with v1.13.6.

---

## Overview

This release fixes MongoDB compatibility gaps:

- **`$collStats` aggregation stage**: Added full support for collection statistics aggregation with count, size, storageSize, totalIndexSize, and indexSizes metrics
- **`collStats` command**: Enhanced to return actual statistics instead of placeholder zeros
- **Aggregation pipeline parsing**: Fixed handling of stage names with leading/trailing whitespace

---

## Bug Fixes

### Fixed: `$collStats` Aggregation Stage Support

**Problem**: NeoSQLite did not support the `$collStats` aggregation stage, which is used to retrieve collection statistics in MongoDB.

**After (MongoDB-compatible)**:

```python
pipeline = [{"$collStats": {}}]
result = collection.aggregate(pipeline)
stats = list(result)[0]
# Returns: {
#     "ns": "database.collection",
#     "count": 100,
#     "size": 12345,
#     "avgObjSize": 123.45,
#     "storageSize": 40960,
#     "totalIndexSize": 8192,
#     "indexSizes": {"_id_": 8192}
# }
```

**Options Support**:

```python
# Count only
collection.aggregate([{"$collStats": {"count": {}}}])
# Returns: {"count": 100}

# Storage stats
collection.aggregate([{"$collStats": {"storageStats": {}}}])
# Returns: {"ns": "...", "storageStats": {...}}
```

### Fixed: `collStats` Command Statistics

**Problem**: The `collStats` wire protocol command returned zeros for size-related fields.

**Before (incorrect)**:

```python
{
    "ok": 1,
    "count": 100,
    "size": 0,              # Wrong
    "storageSize": 0,       # Wrong
    "totalIndexSize": 0,    # Wrong
    "indexSizes": {}        # Wrong
}
```

**After (correct)**:

```python
{
    "ok": 1,
    "count": 100,
    "size": 12345,          # Actual document size
    "avgObjSize": 123.45,   # Average object size
    "storageSize": 40960,   # Actual storage size
    "totalIndexSize": 8192, # Total index size
    "indexSizes": {"_id_": 8192}  # Per-index sizes
}
```

### Fixed: Aggregation Stage Name Parsing

**Problem**: Stage names with leading or trailing whitespace (e.g., `" $collStats"`) were not properly handled, causing "stage not supported" errors.

**Before (could fail)**:

```python
# This would fail with "Aggregation stage ' $collStats' not supported"
collection.aggregate([{" $collStats": {}}])
```

**After (quoted and stripped)**:

```python
# Now works correctly - stage names are stripped before matching
collection.aggregate([{" $collStats": {}}])  # Works!
```

## Migration Guide

### Upgrading from v1.13.6

**No breaking changes** — v1.13.7 is fully backward compatible.

### If You Use `$collStats` Aggregation

You can now use `$collStats` in aggregation pipelines:

```python
# Get full collection statistics
stats = list(collection.aggregate([{"$collStats": {}}]))[0]
print(f"Count: {stats['count']}")
print(f"Size: {stats['size']} bytes")
print(f"Storage: {stats['storageSize']} bytes")

# Get count only
count_result = list(collection.aggregate([{"$collStats": {"count": {}}}]))[0]
print(f"Count: {count_result['count']}")
```

### If You Use `collStats` Command via Wire Protocol

The command now returns accurate statistics:

```python
# Via PyMongo (through NX-27017)
db.command("collStats", "collection_name")
# Now returns actual size metrics instead of zeros
```

## Known Limitations

### NX-27017 Limitations

- **No replication/sharding**: Coming never™ — This is NX-class, not NCC-1701!
- **Change streams via replica set**: Not supported (SQLite-trigger-based `watch()` is available!)
- **`find_raw_batches` with batch_size**: Requires cursor state management
